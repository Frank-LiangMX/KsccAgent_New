# GenericAgent 对齐任务清单

最后更新：2026-05-06

## 目标说明

在不推翻当前 `KsccUI`（PyQt 桌面产品形态）的前提下，对齐 GenericAgent 的核心能力：

- 更强的任务执行循环（计划 -> 执行 -> 反思 -> 修正）
- 更强的分层记忆与任务技能沉淀
- 更强的可观测性与自动化能力（同时保持可控和安全）

## 与 GenericAgent 仍有差距（Gap）

### 1) 执行内核差距

- 缺少结构化策略循环：计划/执行/反思/再计划的标准状态机
- 缺少失败恢复策略：自动重试、分支探索、失败分类
- 缺少任务级运行图（每一步输入输出与决策理由）

### 2) 记忆与技能进化差距

- 缺少自动 SOP 晶化流水线（质量评估 + 自动上架/降级）
- 缺少 insight 索引层（可检索的跨任务经验索引）
- 缺少按任务类型动态注入（当前偏统一注入）

### 3) 工具与自动化场景差距

- 缺少设备级自动化（ADB 等）统一适配层
- 缺少网页/系统自动化的安全审批策略分层
- 缺少高风险任务（支付/下单）专用防误触流程

### 4) 可观测性与评估差距

- 缺少系统化指标：任务成功率、tool 错误率、skill 贡献率
- 缺少回放/审计视图（任务时间线、关键步骤截图/证据）
- 缺少回归基准集（同任务跨模型/跨版本稳定性）

## 完全实现 GenericAgent 功能的任务拆分（Roadmap）

> 说明：以下按“收益 / 风险 / 实施顺序”拆分，建议逐阶段验收，不做一次性大改。

## Phase 1：执行内核对齐（高优先）

目标：让 Agent 从“单轮对话驱动”升级为“任务状态机驱动”。

- [ ] P1-1 定义统一 TaskState 结构（plan/steps/results/reflection/errors）
- [ ] P1-2 实现执行循环状态机（Plan -> Execute -> Reflect -> Next）
- [ ] P1-3 增加失败恢复策略（重试/降级/改写子任务）
- [ ] P1-4 增加每步运行日志（可追踪每一步输入输出）
- [ ] P1-5 UI 增加任务步骤视图（最小版）

验收标准：

- 同一复杂任务可自动分步完成
- 出错后可自动恢复至少一次
- 用户可看到清晰的步骤轨迹

## Phase 2：Skill 晶化自动化（高优先）

目标：从“人工保存 skill 草稿”升级为“半自动 -> 自动沉淀闭环”。

- [ ] P2-1 Skill 草稿质量评分（完整性、复用性、成功信号）
- [ ] P2-2 自动候选入库策略（阈值 + 人工兜底）
- [ ] P2-3 Skill 退化与淘汰机制（低命中/低收益）
- [ ] P2-4 Skill 冲突仲裁（相似技能聚合）
- [ ] P2-5 增加 Skill 效果统计面板

验收标准：

- Skill 自动增长且无明显污染
- 命中收益可量化（耗时下降、成功率提升）

## Phase 3：分层记忆增强（高优先）

目标：补齐 GenericAgent 风格的层次化记忆检索与注入调度。

- [ ] P3-1 新增 Insight Index 层（跨任务可检索摘要）
- [ ] P3-2 任务类型识别 -> 选择性注入策略
- [ ] P3-3 记忆压缩策略（短中长期不同规则）
- [ ] P3-4 记忆冲突检测（过期事实/矛盾事实）
- [ ] P3-5 记忆命中可视化（本轮命中来源）

验收标准：

- 长会话 token 成本可控
- 相关记忆命中率明显提升

## Phase 4：工具能力与安全策略（中高优先）

目标：对齐示例场景能力，同时确保桌面产品安全可控。

- [ ] P4-1 抽象 Tool Capability 层（读写、浏览、设备、外部 API）
- [ ] P4-2 高风险动作审批策略（多级确认 + 白名单）
- [ ] P4-3 网页自动化与页面证据采集（截图/日志）
- [ ] P4-4 设备自动化（ADB）适配与隔离
- [ ] P4-5 敏感任务风控模板（支付/下单类）

验收标准：

- 自动化能力可用且可解释
- 高风险操作默认可拦截

## Phase 5：评估与产品化（中优先）

目标：让“能力提升”可被度量、回归、长期维护。

- [ ] P5-1 建立标准任务基准集（10~30 个）
- [ ] P5-2 自动回归脚本（跨后端/跨模型）
- [ ] P5-3 指标看板（成功率、耗时、token、错误率）
- [ ] P5-4 会话审计视图（模型切换、关键事件）
- [ ] P5-5 发布规范（实验特性开关与迁移策略）

验收标准：

- 每次改动都有可量化前后对比
- 回归失败可快速定位
